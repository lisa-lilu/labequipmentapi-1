================================================================================
EDGE CASE 10: Delete Category with Equipment (409 Conflict)
Expected: 409 Conflict
Cannot delete category with dependent equipment
================================================================================

# Create a category
curl -X POST http://localhost:3000/categories \
  -H "Content-Type: application/json" \
  -d '{"category": {"name": "Electronics"}}'

# Note the category ID (e.g., CATEGORY_ID_2)
# Create equipment for that category
curl -X POST http://localhost:3000/equipment \
  -H "Content-Type: application/json" \
  -d '{
    "equipment": {
      "name": "Oscilloscope",
      "serial_number": "OSC001",
      "status": "available",
      "category_id": CATEGORY_ID_2
    }
  }'

# Try to delete the category (should fail)
curl -X DELETE http://localhost:3000/categories/CATEGORY_ID_2 \
  -H "Content-Type: application/json"

Response should show:
{
  "error": "Cannot delete category. 1 equipment items still belong to it."
}
HTTP Status: 409 Conflict

CATEGORY VALIDATIONS:
✓ Name is required (presence: true)
✓ Name must be at least 3 characters (length: { minimum: 3 })
✓ Name must be unique (uniqueness: true)
✓ Cannot delete category with equipment (dependent: :restrict_with_error)

✓ Serial number is required (presence: true)
✓ Serial number must be unique (uniqueness: true)
✓ Serial number must be alphanumeric only (custom validation)
✓ Status must be one of: available, in_use, maintenance
✓ Category ID is required (belongs_to: :category)

MAINTENANCE RECORD VALIDATIONS:
✓ Description is required (presence: true)
✓ Performed_at is required (presence: true)
✓ Performed_at cannot be in the future (custom validation)
✓ Equipment ID is required (belongs_to: :equipment)

HTTP STATUS CODES:
✓ 200 OK - Successful GET/PATCH
✓ 201 Created - Successful POST
✓ 204 No Content - Successful DELETE
✓ 404 Not Found - Resource not found
✓ 409 Conflict - Cannot delete category with equipment
✓ 422 Unprocessable Entity - Validation failed

1. Start the Rails server:
